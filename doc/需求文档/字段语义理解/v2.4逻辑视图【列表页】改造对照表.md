# 一、逻辑视图【列表页】改造对照表

> 定位：**表级语义治理入口 + 调度台**
> 原则：**保留结构，替换语义，不加复杂交互**

---

## 1️⃣ 页面角色对照

| 项    | 现状        | 调整后                |
| ---- | --------- | ------------------ |
| 页面角色 | 元数据 / 表清单 | **语义治理入口（表级）**     |
| 用户心智 | “看看有哪些表”  | **“我现在要从哪些表开始治理”** |
| 是否保留 | —         | ✅ 完整保留             |

---

## 2️⃣ 状态字段对照（非常关键）

| 现状状态（示例）    | 问题       | 调整后状态（统一口径） |
| ----------- | -------- | ----------- |
| 未分析 / AI处理中 | 系统态，用户不懂 | 未开始语义建模     |
| 有建议         | 不知道下一步   | 字段语义待确认     |
| 已分析         | 模糊       | 语义建模进行中     |
| 通过          | 太技术      | 可进入对象建模     |

> **规则**：
> 列表页的状态 = **语义建模阶段态**，不是系统执行态。

---

## 3️⃣ CTA（主行动按钮）对照

| 现状           | 问题      | 调整后              |
| ------------ | ------- | ---------------- |
| 无 CTA / 多个操作 | 不知道先点哪个 | **每行 1 个主 CTA**  |
| 表名点击         | 行为不明确   | 表名 = 查看，CTA = 行动 |

### CTA 规则（直接定死）

| 当前状态    | CTA 文案   | 行为             |
| ------- | -------- | -------------- |
| 未开始语义建模 | 开始字段语义理解 | 打开详情页 → 语义理解模式 |
| 字段语义待确认 | 继续字段语义理解 | 打开详情页 → 语义理解模式 |
| 可进入对象建模 | 查看语义结果   | 打开详情页 → 浏览模式   |

---

## 4️⃣ 批量操作区对照

| 现状          | 调整后             |
| ----------- | --------------- |
| 批量生成建议      | **批量生成字段语义建议**  |
| 批量任务执行后状态不清 | 执行后状态 → 字段语义待确认 |

> ❌ 列表页不支持裁决
> ❌ 列表页不支持字段级编辑

---

## 5️⃣ 质量 / 安全信息对照

| 项  | 现状     | 调整后             |
| -- | ------ | --------------- |
| 质量 | 多个指标   | 保留，但明确为**只读概览** |
| 安全 | 等级/策略  | 保留，但无编辑入口       |
| 行为 | 可能跳转编辑 | **只跳转对应域页面**    |

---

# 二、逻辑视图【详情页】改造对照表

> 定位：**字段语义治理的上下文承载页**
> 原则：**同一页面，双模式，不跳转**

---

## 1️⃣ 页面整体结构

| 区域     | 现状                   | 调整后            |
| ------ | -------------------- | -------------- |
| 页面 URL | /logical-view/detail | **不变**         |
| 页面布局   | 左字段 + 中上下文           | **保留**         |
| 新增概念   | 无                    | **页面模式（Mode）** |

---

## 2️⃣ 页面模式对照（核心）

| 模式     | 现状 | 调整后    |
| ------ | -- | ------ |
| 浏览模式   | 默认 | 保留（原样） |
| 语义理解模式 | 无  | **新增** |

### 模式切换入口

* 列表页 CTA 进入 → 默认语义理解模式
* 详情页按钮 → “进入字段语义理解”

---

## 3️⃣ 顶部区域对照

| 现状        | 调整后                    |
| --------- | ---------------------- |
| 表名 / 基础信息 | 保留                     |
| 无治理信息     | **新增语义建模进度条（仅语义理解模式）** |

**示例：**

```
t_hr_employee
字段语义理解中｜已确认 12/30｜阻塞 2
```

---

## 4️⃣ 左侧字段列表对照

| 现状    | 调整后                |
| ----- | ------------------ |
| 字段名   | 字段名                |
| 类型    | 类型                 |
| 无治理状态 | **新增：语义状态标记**      |
| 无风险提示 | **新增：风险/置信度 icon** |

> 这是“字段治理列表”，不是普通字段列表。

---

## 5️⃣ 中部字段上下文对照（重点：强复用）

| 项       | 现状 | 调整后 |
| ------- | -- | --- |
| 类型      | 保留 | 不动  |
| 示例值     | 保留 | 不动  |
| Profile | 保留 | 不动  |
| 统计信息    | 保留 | 不动  |

> **这里尽量 0 改动**，保证连续感。

---

## 6️⃣ 右侧区域对照（唯一新增重块）

| 现状 | 调整后                   |
| -- | --------------------- |
| 无  | **语义裁决面板（语义理解模式下显示）** |

### 裁决面板内容（定版）

* 字段角色（Role）【必选】
* 业务含义（Term）【可选】
* 语义标签（Tags）【可选】
* 建议与证据（规则命中 / AI 解释）
* 操作：接受 / 修改 / 拒绝

---

## 7️⃣ 完成态对照

| 现状     | 调整后          |
| ------ | ------------ |
| 无明确结束  | **自动识别完成态**  |
| 无下一步引导 | CTA：进入候选业务对象 |

---

# 三、哪些地方【明确不改】

为了防止返工，这些点建议你**明确写进 PRD**：

* ❌ 不新增“字段语义理解详情页”
* ❌ 不在逻辑视图页编辑质量规则 / 安全策略
* ❌ 不在列表页裁决字段语义
* ❌ 不复制字段上下文 UI

---

# 四、给你一个“最终统一说法”（可直接用）

> 逻辑视图列表页是**语义建模的表级入口与调度台**；
> 逻辑视图详情页是**字段语义理解发生的上下文容器**；
> 字段语义理解不是新页面，而是详情页的一种工作模式。
