# 高置信度识别弹窗优化（20260117-07）

## 目标与范围
- 约束“直接生成业务对象”的越权风险，受 Gate 与待Review 前置校验。
- 明确“直接生成”与“加入候选”的行为差异，避免误操作。

## 需求要点（P0）
1. 直接生成必须受 Gate 与待Review 约束
- 在弹窗中展示“生成前置校验清单”。
- 任一校验未通过 → 直接生成置灰，仅允许加入候选。

2. 明确两者差异
- 直接生成＝写入语义注册表（生效，可回滚）。
- 加入候选＝草稿对象（不生效，待Review）。

## 生成前置校验清单
- 主键是否已确认
- 生命周期要素是否通过 Gate
- 敏感冲突是否已处理
- 影响范围是否可接受（待Review=0）

## UI 结构
- 顶部：表名 + 置信度 + 业务描述
- 中部：生成前置校验清单（通过/未通过）
- 提示：动作差异说明
- 底部：
  - 直接生成业务对象（未通过置灰）
  - 加入候选业务对象

## 交互规则
- 未通过校验时，直接生成按钮置灰并给出提示。
- 直接生成时再次校验，未通过则阻断并提示。
