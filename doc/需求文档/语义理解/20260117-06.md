# 语义理解审校摘要与字段 Review 优化（20260117-06）

## 目标与范围
- 解决审校摘要中 Gate 与 Score 口径冲突问题，统一评分尺度到 0-100。
- 支持策略模板切换并展示差异，减少“策略不匹配”的理解成本。
- 字段结构页补齐批量 Review 工作台与融合结果状态标识。

## 一、语义理解引擎审校摘要优化（P0）

### 1. 关键要素完整度与 Gate 口径统一
- 完整度必须基于 Gate 结果计算。
- 若生命周期 Gate 未通过，完整度显示为“缺失（需补齐）”。
- 同时列出缺失原因（主键/生命周期/表类型）。

### 2. 评分尺度统一 + Gate/Score 分区展示
- 所有评分统一为 0-100。
- Gate（硬拦截）与 Score（成熟度）物理分区：
  - Gate：通过/不通过 + 修复任务。
  - Score：用于排序与持续改进，不直接阻断。

### 3. 策略模板可切换 + Diff
- 支持模板切换：ODS / DWD / DIM / DWS / 快照。
- 切换后显示：切换前后评分变化与影响因子 Top3。
- 展示“系统推荐策略”作为基线。

### UI 结构（审校摘要）
- 审校摘要三维指标：覆盖率 / 关键要素完整度 / 风险项
- Gate 区：状态 + 修复任务
- Score 区：表维度得分 / 字段维度得分
- 评分透视：策略切换下拉 + 差异说明 + 评分拆解表

## 二、字段结构页优化（理解后）

### 1. 批量 Review 工作台（P0）
- 勾选字段后提供批量动作：
  - 批量接受高置信
  - 批量改判
  - 加入待办
- 提供分组视图：
  - Group A：可批量
  - Group B：需下钻
  - Group C：高风险

### 2. 融合结果状态标识
- 融合结果旁新增状态：建议 / 已确认 / 待定。
- 只有“已确认”状态允许生成业务对象与问数资产。

### UI 结构（字段页）
- 顶部：搜索 + 视图切换 + 分组筛选
- 批量工作台条：全选当前筛选 / 已选数 / 高置信数 / 批量动作
- 字段表格：新增选择列 + Group 标签 + 融合结果状态标签

## 规则与交互说明
- Group A（可批量）：高置信 + 低敏感（L1/L2）且无冲突。
- Group C（高风险）：L3/L4 或低置信或冲突。
- 批量接受高置信仅对 Group A 生效，其余保留待定。
- 批量改判将字段标记为“待定”，进入编辑态。
- 批量动作写入审计日志（字段/动作/时间/原因）。
