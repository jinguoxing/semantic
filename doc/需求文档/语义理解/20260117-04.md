# 理解后概览页优化方案（状态/可定位/数据驱动摘要）

日期：2026-01-17

## 目标
- 消除“语义理解完成 = 治理生效完成”的认知误导（P0）。
- 让“待处理/门槛复核”具备可点击定位与可操作入口（P0）。
- 摘要与指标一致，禁止泛化话术（P0）。
- 将“确认排除”改为可逆治理标记（P1）。

## 现状问题（必须改）
1) 顶部状态使用“语义理解完成”，用户误认为已生效。
2) 待处理/门槛复核只有提示，无法定位问题字段或规则。
3) 摘要文案与实际指标不一致风险高。
4) “确认排除”强动作位置显眼且不可逆，认知风险高。

## 优化方案（定版）

### 1) 状态表达统一为三段式（P0）
- 已生成建议（待Review）
- 已确认生效
- 已回滚

页面文案：
- S1：显示“建议已生成，待确认”
- S3：显示“已确认生效”
- 回滚：显示“已回滚（可恢复）”

要求：状态在列表页、详情页、字段视图统一呈现。

### 2) 待处理/门槛复核可点击定位（P0）
- 点击“待处理 n”
  - 跳转字段列表
  - 自动过滤到“问题字段”
  - 支持冲突/敏感未确认/低置信三类标签
- 点击“门槛需复核”
  - 展开 Gate 明细卡片
  - 展示未通过项、原因与推荐操作

### 3) 摘要文案数据驱动（P0）
摘要必须由实际指标生成，且与计数一致。

模板（示例）：
- Gate 未通过项（n条）：列出 1~3 条原因
- 风险项（n条）：列出 1~3 条原因
- 待Review（n条）：字段 n / 对象 n

规则：
- 若某类为 0，则不显示该段
- 禁止出现“建议先处理冲突字段”但冲突数=0 的文案

### 4) 排除动作改为可逆标记（P1）
- “确认排除”改为“标记为不纳入治理范围”
- 需填写：原因/责任人/可恢复说明
- 放在次级操作区（非主操作）
- 与回滚/版本机制一致，可追溯

# UI 结构与交互稿（概要）

## 一、页面顶部状态区（替换“语义理解完成”）

结构：
- 状态徽章（S1/S3/回滚）
- 主说明文案
- 次级说明（显示 Review/回滚说明）

示例：
- S1：徽章“已生成建议（待Review）” + 文案“建议已生成，待确认”
- S3：徽章“已确认生效” + 文案“该建议已纳入治理”
- 回滚：徽章“已回滚” + 文案“已回滚，可恢复”

交互：
- 状态徽章可点击查看状态流转历史（可选）

## 二、摘要区（数据驱动）

结构：
- 三段可折叠摘要卡片
  1. Gate 未通过项（n条）
  2. 风险项（n条）
  3. 待Review（n条）

每段结构：
- 标题 + 计数
- 1~3 条摘要原因
- “查看详情”按钮

交互：
- 点击 Gate 未通过项 → 展开 Gate 明细
- 点击 待Review → 跳转字段列表并自动过滤
- 点击 风险项 → 跳转问题字段并按风险标签过滤

## 三、Gate 明细卡片（门槛复核）

结构：
- 未通过项列表（主键/生命周期/表类型）
- 原因说明
- 推荐操作（补主键/补生命周期/人工确认）

交互：
- 与摘要区联动（点击摘要自动展开）

## 四、字段问题定位

结构：
- 字段列表顶部增加“问题字段过滤器”
  - 冲突
  - 敏感未确认
  - 低置信

交互：
- 从摘要区点击进入自动开启对应过滤
- 可一键清除过滤恢复全量

## 五、排除动作区（改为可逆标记）

结构：
- 放在次级操作区/更多菜单
- 文案：标记为不纳入治理范围
- 弹层字段：原因、责任人、可恢复说明

交互：
- 标记后显示“已标记”状态，可撤销

# 数据对齐（口径）

## 待Review
- 字段级冲突
- L3/L4 未确认
- 置信度低于阈值

## Gate 未通过
- Gatekeeper REJECT/REVIEW
- 缺失主键/生命周期
- 命中排除规则

## 风险项
- 分数低于阈值
- 敏感字段占比高
- 冲突数量超阈值

---

# 验收标准
- 顶部不再出现“语义理解完成”字样
- 待处理/门槛复核均可点击定位
- 摘要文案与数据严格一致
- 排除动作改为可逆标记
