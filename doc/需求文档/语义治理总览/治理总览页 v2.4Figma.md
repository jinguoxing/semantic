# 治理总览页 · Figma Frame 结构说明（v2.4）

---

## Frame 0｜Page Container（页面级）

```text
Frame: GovernanceOverviewPage
- Width: 1440
- Padding: 24
- Background: #F7F8FA
```

**包含子 Frame（自上而下）：**

1. Header / Filters
2. KPI Summary
3. Governance Pipeline
4. Blockers & Risks
5. Governance Distribution
6. Quick Actions
7. Active Semantic Version

---

## Frame 1｜Header & Filters（页面头部）

```text
Frame: HeaderBar
- Height: 64
- Layout: Horizontal (Space-between)
```

### 左侧

```text
Text: 治理总览
SubText: 当前语义版本下的治理状态总览
```

### 右侧

```text
SegmentedControl: 时间范围
- 近7天 | 近30天（默认） | 近90天

StatusTag:
- 协同运行中 / 治理阻塞中
```

---

## Frame 2｜KPI Summary（治理 KPI 总览）⭐ 第一屏核心

```text
Frame: GovernanceKPIGroup
- Layout: Grid (4 columns)
- Gap: 16
```

### KPI Card（统一组件）

```text
Component: GovernanceKPICard
- Title
- MainValue
- SubText
- StatusColor (green / yellow / red)
- Clickable
```

#### KPI-1 字段语义覆盖率

```text
Title: 字段语义覆盖率
Value: 68%
Sub: 已确认字段 / 总字段
```

#### KPI-2 数据质量 Gate

```text
Title: 质量 Gate 通过率
Value: 82%
Sub: 阻塞表 5
```

#### KPI-3 数据安全分级

```text
Title: 安全分级完成率
Value: 74%
Sub: 高敏字段 3 未处理
```

#### KPI-4 治理风险等级

```text
Title: 当前治理风险
Value: 中风险
Sub: 基于阻塞项自动评估
```

---

## Frame 3｜Governance Pipeline（治理主线进度）

```text
Frame: GovernancePipeline
- Layout: Vertical
```

### 标题

```text
Text: 治理主线进度
```

### Pipeline Component

```text
Component: PipelineStepBar
Steps:
1. 逻辑视图
2. 字段语义理解
3. 数据质量
4. 数据安全
5. 语义版本
```

每个 Step 显示：

* 状态：未开始 / 进行中 / 已完成
* Count Badge（如：12 个待确认字段）
* 当前瓶颈 Step 高亮

---

## Frame 4｜Blockers & Risks（治理阻塞项）⭐ 指挥核心

```text
Frame: GovernanceBlockers
- Layout: Grid (3 columns)
- Gap: 16
```

### Blocker Card（统一结构）

```text
Component: BlockerCard
- Icon
- Title

- CTA: 去处理
```


## 组件复用建议（给前端）

你这页至少可以沉淀 6 个通用组件：

1. GovernanceKPICard
2. PipelineStepBar
3. BlockerCard
4. DistributionCard
5. ActionRow
6. VersionHighlightCard

👉 **后续字段 / 质量 / 安全页面都能复用**

---

## 最后一句定位确认（非常重要）

> **这张页不是“建模进度面板”，
> 而是“当前语义版本是否可安全使用”的治理仪表盘。**
