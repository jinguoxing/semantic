## B 区｜治理主线进度（状态感知）

### 📌 你现在的问题

* 用「业务视角 / 技术视角」切
* 用户不知道卡在哪一步

---

### ✅ 改成「治理主线进度条」

```text
治理主线进度
[ 逻辑视图 ] → [ 字段语义理解 ] → [ 数据质量 ] → [ 数据安全 ] → [ 语义版本 ]
```

每一段显示：

* 未开始 / 进行中 / 已完成
* 当前瓶颈步骤高亮

📌 示例：

```text
字段语义理解：进行中（12 个待确认字段）
```

## C 区｜治理阻塞与风险（必须优先处理）

### 📌 这是你现在页面里**最有价值的一块**，但命名错了

---

### ❌ 当前命名

* 业务阻点
* 治理阻点

### ✅ 统一为一个区块

#### 标题

```text
治理阻塞项（需处理）
```

#### 内容结构（3 类，固定）

```text
语义阻塞
- 关键字段未确认（2）

质量阻塞
- Gate 未通过表（5）

安全风险
- 高敏字段未分级（3）
```

📌 规则：

* 每条都能点
* 直达字段语义理解 / 质量 / 安全页面

## 🚫 明确：哪些模块要移出治理总览页

请**整体下沉或移走**：

* 协同成熟度指数
* 业务目标 / KPI 口径覆盖
* 协同流程链示意图

👉 它们适合：

* 协同分析页
* 方法论展示页
* 管理汇报页
  **不适合治理指挥台**
