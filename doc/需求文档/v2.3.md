# 企业级 AI 语义治理平台设计规范 (V2.3 终极版)

## 核心理念变更总结

1. **从诊断到行动 (Actionable Insights):** 不再只告诉用户“哪里错了”，而是直接提供“一键修复”的按钮。
2. **游戏化治理驱动 (Gamification):** 明确告知用户每项治理动作带来的收益（置信度提升），激励用户完成任务。
3. **严谨的统计口径 (Rigorous Metrics):** 建立互斥、清晰的摘要统计标准，杜绝数据打架。
4. **可控的决策闭环 (Controllable Loop):** 术语可沉淀复用，高风险操作可预览、可回滚。

* 点击 **[🔗 指定语义主键]**：页面滚动/跳转到详情页的字段列表，高亮引导用户勾选标识符。
* 点击 **[🔍 识别 JSON 结构]**：触发 AI 对该字段进行采样分析，尝试解析其 Schema。

### 2. 改造：术语命名的最小闭环 (Term Loop)

**UI 设计规范：**

* **业务名称输入框**：改为带联想功能的输入框 (AutoComplete)。
* **交互流程**：
1. **AI 冷启动推荐**：输入框预填灰色文字 “用户基础信息表 (AI建议)”。
2. **复用已有术语**：当用户点击输入框开始输入时，优先下拉展示标准术语库中已存在的词条，并打上 **[标准术语]** 标签。
3. **沉淀新术语**：如果用户输入了一个全新的名称并点击确认，系统默默将此名称记入轻量级术语表，供下次相似表扫描时推荐。

### 2. 改造：可预览、可回滚的升级建议 (Upgrade Workflow)

针对 AI 提出的结构化升级建议（如“升级为行为对象”），提供完整的决策支持。

**UI 设计规范 (升级建议卡片)：**
