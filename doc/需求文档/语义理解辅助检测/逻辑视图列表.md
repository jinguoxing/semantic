

# Figma 设计稿

## 「语义理解辅助模板说明」弹窗 / 抽屉

> 触发方式：
> **在批量语义理解弹窗中，点击「SEMANTIC_MIN」或 ℹ️ 图标**

---

## Frame F1

### Semantic Assist Template — Overview（模板说明）

### 1️⃣ 弹窗形态（推荐）

* 类型：**Side Drawer（右侧抽屉）** 或 Modal（二选一）
* 宽度：480–560px
* 关闭方式：右上角 × / Esc

> 推荐抽屉：
> 不打断批量配置流程，语义更像“说明文档”。

---

### 2️⃣ Header 区

```text
语义理解辅助模板

SEMANTIC_MIN
```

右侧可有 Badge：

```text
默认模板 · v2.4
```

---

### 3️⃣ 模板定位说明（必须有，放在最上面）

**Info Card（浅色背景）**

```text
SEMANTIC_MIN 是语义理解阶段的默认辅助模板，
用于为语义建议提供数据分布与一致性信号。

• 仅作为语义理解辅助信号
• 不产生质量通过/失败结论
• 不影响语义裁决与语义版本生效
```

---

## 4️⃣ 模板内容区（核心）

> 用 **“信号维度”**，而不是“规则”这个词
> 这是避免误解的关键

---

### 4.1 字段基础一致性信号

**Section Title**

```text
字段一致性信号
```

**表格 / List 结构（只读）**

| 信号名称   | 含义          | 语义用途        |
| ------ | ----------- | ----------- |
| 空值比例   | 字段中空值占比     | 判断字段是否稳定    |
| 去重比例   | 不同值占比       | 判断是否可能为 ID  |
| 类型可解析率 | 可被目标类型解析的比例 | 判断时间 / 数值字段 |

> 底部小字说明：

```text
以上信号优先复用已有质量检测结果，不会重复执行质量规则
```

---

### 4.2 字段值分布信号（语义专用）

**Section Title**

```text
字段分布信号
```

| 信号名称  | 含义           | 语义用途        |
| ----- | ------------ | ----------- |
| 常见值分布 | 字段最常见的 Top 值 | 判断状态 / 枚举字段 |
| 值集中度  | 常见值占比        | 判断状态是否稳定    |

> 辅助说明（灰字）：

```text
该类信号通常不包含在质量检测中，仅用于语义理解
```

---

### 4.3 范围与异常提示（可选）

**Section Title**

```text
范围与异常提示
```

| 信号名称      | 含义              |
| --------- | --------------- |
| 数值 / 时间范围 | 判断字段是否符合时间或数值语义 |
| 异常时间提示    | 未来时间 / 极端时间值    |

---

## 5️⃣ 风险提示机制说明（避免误解）

### Section：风险提示如何使用？

```text
模板中部分信号会生成风险提示（Risk Flags），
用于提醒语义建议可能存在不稳定或歧义。

• 风险提示仅用于解释与提醒
• 不等同于数据质量问题
• 不会阻止语义理解流程
```

可用 Tag 示例（只读）：

```text
HIGH_NULL   LOW_UNIQUENESS   ENUM_NOT_STABLE
```

---

## 6️⃣ 明确的「不可配置」声明（非常重要）

### Section：模板使用说明（底部）

**Warning / Note 样式**

```text
当前模板内容为平台内置默认能力：

• 不支持编辑或删除信号
• 不支持调整阈值或规则
• 不影响数据质量模块中的任何规则配置
```

---

```text
✓ 使用模板：SEMANTIC_MIN
✓ 采样比例：1%

```

---

# 设计原则总结（你可以直接跟设计师说）

> 这是一个**模板说明视图，不是规则配置视图**。
> 只解释“包含什么信号、做什么用”，
> 不出现“规则、阈值、校验、通过/失败”等词。

---

# 最终你这个交互的产品判断

✔ 允许用户“看到模板内容”
✔ 不引入质量规则配置认知
✔ 不影响批量流程
✔ 为后续 SEMANTIC_STD / PII 模板扩展留好位置

这是一个**非常成熟的企业级产品交互**。
