# BizSemantic v2.3

## 业务语义建模平台 · 冻结版方法论（一页式）

---

## 1. 核心目标（一句话）

> **让数据具备稳定、可解释、可治理的业务语义，
> 通过 AI + 规则 + 人机协作，从底层数据中形成可信的业务对象。**

---

## 2. 核心原则（不可违反）

1. **语义不是自动生效的**

   * AI 和规则只“建议”，**用户确认是唯一生效点**
2. **先理解数据，再抽象业务**

   * Bottom-up 是主路径，Top-down 仅用于补充
3. **规则管边界，AI 管语义，人管决策**
4. **所有结论必须可解释、可回溯**
5. **不要求用户懂建模，只让用户做业务判断**

---

## 3. 对象模型（冻结定义）

平台只承认三类语义对象（内部）：

| 内部模型     | 对外业务语言 | 含义                 |
| -------- | ------ | ------------------ |
| Entity   | 业务对象   | 现实中稳定存在的东西（如用户、订单） |
| Event    | 业务过程   | 一次性发生的事情（如支付、审批）   |
| Snapshot | 业务结果   | 某一时刻的状态结论（如订单状态）   |

> UI **不暴露** Entity / Event / Snapshot 术语。

---

## 4. 语义分层（L0–L3）

* **L0 现实业务事实**：人、事、状态（用户输入 / 访谈）
* **L1 业务对象**：统一语义对象（平台核心资产）
* **L2 使用场景**：分析 / 治理 / 问数语境
* **L3 数据映射**：对象 ↔ 系统 / 表 / 字段

> **业务对象在 L1 唯一，允许多 L3 映射**

---

## 5. Bottom-up 语义理解流程（主路径）

```
数据源连接
 → 元数据扫描
 → 规则 Gate（排除必错表）
 → 规则评分 + AI 语义理解
 → 业务对象 / 字段语义 / 升级建议（候选）
 → 结果对比 & 冲突解释
 → 用户确认
 → 语义资产入库（L1/L3）
```

---

## 6. AI + 规则混合判定策略（冻结）

* **规则负责**

  * 日志/临时/关系表排除
  * 技术字段识别
  * 主键 / 生命周期 / 枚举结构
* **AI 负责**

  * 表/字段语义理解
  * 业务命名与解释
  * 歧义判断
* **合成机制**

  * Rule Score + AI Score → Final Score
  * 阈值决定 Auto / Review / Reject
* **冲突处理**

  * 硬规则覆盖
  * 否则进入人工确认

---

## 7. 评分模板体系（场景化体检）

* 不同表类型使用不同评分模板：

  * OLTP / ODS / DWD / DWS / 维表 / 快照 / 日志
* 模板可：

  * 系统自动推荐
  * 用户手动切换
* 切换模板 ≠ 改变事实，只改变评判标准
* UI 必须展示：

  * 当前模板
  * 推荐原因
  * 切换前后得分 Diff

---

## 8. 字段语义与统计口径（冻结）

* 字段语义 **互斥统计**（摘要用）：

  * 标识符 > 生命周期 > 状态 > 业务属性
* 详情页允许：

  * Primary Role + Secondary Tags
* 摘要、字段列表、敏感统计 **口径必须一致**

---

## 9. 状态 / 行为升级原则（防对象膨胀）

* 升级为 **建议**，不自动生效
* 必须支持：

  * 升级预览
  * 接受 / 拒绝 / 稍后
  * 回滚
* 升级前必须展示影响面（Blast Radius）

---

## 10. 敏感数据原则（强一致）

* 字段级识别 → 表级聚合（max / exists）
* 表级安全等级 = 字段最高等级
* UI 与 API 权限展示必须一致
* 所有敏感结论必须可解释（规则 / AI 依据）

---

## 11. 术语能力定位（冻结）

* **不做独立术语建模**
* 术语是 **业务对象的属性**
* 每个术语必须有：

  * 作用域（GLOBAL / DEPT / SCENARIO / SYSTEM）
  * 首选名（per scope）
  * 来源与置信度
* 术语用于：

  * 搜索 / 对齐 / 问数入口
  * 不直接改变语义模型

---

## 12. Run Snapshot（企业级可信）

* 每次运行必须可复现：

  * 输入元数据 Hash
  * 规则版本
  * AI 模型 & Prompt 版本
  * 评分模板
* 支持 Run 对比：

  * 置信度变化
  * 角色变化
  * 规则 / AI 变化原因

---

## 13. 生效与治理（唯一口径）

* **Suggestion → Decision → Semantic Registry**
* 所有变更：

  * 可审计
  * 可回滚
  * 可版本化

---

## 14. 非目标（明确不做）

* 不强制业务人员学习建模
* 不做“先建业务词典再映射数据”
* 不允许 AI 直接修改语义资产
